# 👩 1. คุณแพร — ลูกค้า สั่งส้มตำกินกลางวัน

## บริบท
วันจันทร์ เที่ยง 12:15 น. คุณแพรนั่งทำงานอยู่ออฟฟิศที่อโศก หิวมาก คิดถึงส้มตำร้านพี่ก้อยที่กินบ่อย เลยจะสั่งให้ส่งมาที่ออฟฟิศ

## 🎬 เรื่องเริ่ม

### ขั้นที่ 1: เปิดเว็บร้าน (10 วินาที)

คุณแพรกดลิงก์ที่บันทึกไว้ใน Notes:
```
https://order.somtam-saeb.com
```
หรือถ้ายังไม่ตั้ง custom domain ก็เป็น:
```
http://localhost:3000/r/somtam-zab/main
```

**เห็นอะไรบนหน้าจอ:**
- ภาพ hero สีส้ม-เหลือง (theme ของร้าน)
- โลโก้ "ส้มตำแซ่บ"
- เมนูแบ่งเป็นหมวด: ส้มตำ · ยำ · แกง · ของหวาน · เครื่องดื่ม
- รายการแนะนำ (Chef's recommendation) มี badge ดาว
- ราคาเป็น ฿บาท

### ขั้นที่ 2: เลือกของ (1 นาที)

**กดที่ส้มตำปูปลาร้า ฿80:**
- Popup แสดงรายละเอียด + รูปอาหาร
- ช่อง "Special notes" — คุณแพรพิมพ์ "ไม่ใส่กระเทียม รสจัด"
- กด **+ Add to cart**
- เห็น badge ตะกร้ามุมบนขวาเปลี่ยนเป็น "1"

**เพิ่มอีก:**
- ไก่ทอด ฿120 (ใส่ note "ไม่ต้องน้ำจิ้ม")
- ข้าวเหนียว ฿20 × 2
- น้ำเก๊กฮวย ฿35

**ราคารวมในตะกร้า:** ฿275

### ขั้นที่ 3: เปิดตะกร้า (30 วินาที)

กดที่ icon ตะกร้ามุมบนขวา → ไป `/cart`

**เห็นอะไร:**
- รายการ 4 รายการ พร้อม note ที่เขียน
- มี **🎤 Voice order card** แสดงให้เห็น
- ยอด: ฿275

**ลอง voice order:**
- คุณแพรกดปุ่ม "**Speak**"
- ไมค์เปิด → พูด: "เอาน้ำเปล่าอีก 2 ขวด"
- ระบบประมวลผล 2 วินาที
- เห็น "You said: เอาน้ำเปล่าอีก 2 ขวด"
- ตะกร้าเพิ่มน้ำเปล่า 2 ขวด อัตโนมัติ
- ข้อความเขียว: "Added 2 น้ำเปล่า"

**ยอดใหม่:** ฿305

### ขั้นที่ 4: ไป Checkout (2 นาที)

กดปุ่ม "**Checkout**" ใหญ่สีส้ม

**เห็นฟอร์ม:**

```
📋 ข้อมูลติดต่อ
   ชื่อ: แพร [auto-filled ถ้า sign in]
   เบอร์โทร: 081-234-5678

📍 ที่อยู่จัดส่ง
   [ยังไม่ได้ sign-in หรือไม่มี saved address]
   → input ว่าง พิมพ์ที่อยู่:
   "อโศกทาวเวอร์ ชั้น 15 ถนนสุขุมวิท 21"
```

**ถ้าเคย sign-in มาก่อน:**
จะเห็น radio cards แสดงที่อยู่ที่เคยบันทึก:
```
○ Home (default) — บ้าน หมู่บ้านเดอะลีฟ
● Office — อโศกทาวเวอร์ ชั้น 15
○ Use a new address  (+ พิมพ์ใหม่)
```

**เลือกวิธีจ่ายเงิน:**
```
○ PromptPay   ● Card   ○ Cash
```
คุณแพรเลือก PromptPay (สแกน QR)

**ใส่ Promo code:**
- มี code ที่ร้านส่งใน LINE: `WELCOME10`
- พิมพ์ลงช่อง → กด **Apply**
- ระบบเช็ค... ✅ ได้ส่วนลด 10% (ลด ฿28)
- เห็นกล่องเขียว: "WELCOME10 — saved ฿28"

**Tip slider:**
- 4 ปุ่ม: None / 5% / 10% / 15% / Custom
- คุณแพรเห็นพี่ส่งเข้าทำงานหนัก → กด **10%** = ฿28
- ถ้าจะใส่เป็นบาทตรงๆ ก็พิมพ์ในช่อง custom

**Loyalty redeem:**
- คุณแพรเป็น Silver tier มี 450 points
- Slider ไถได้ตั้งแต่ 0 ถึง 150 (subtotal ×50%)
- เลื่อนไปที่ 100 → ลด ฿100

**สรุปยอด:**
```

Delivery fee:        ฿40
Service fee (5%):    ฿15
Tip:                 ฿28
Promo (WELCOME10):  -฿28
Loyalty discount:  -฿100

```

# 🛵 2. พี่เอก — คนขับวันแรก + วันทำงาน

## วันที่ 1: สมัคร + KYC

### ขั้นที่ 1: เปิดแอพ Driver (1 นาที)

```

```

**เห็น:**
- หน้า login สวยๆ มีรูปวิวมอเตอร์ไซค์
- ช่อง "เบอร์โทร" + ปุ่ม "ส่ง OTP"

พี่เอกใส่ `0812345678` → กดส่ง → รอ SMS

(ในชีวิตจริงต้องตั้ง Twilio ก่อน ไม่งั้น OTP ไม่มา)

### ขั้นที่ 2: ใส่ OTP (10 วินาที)

ใส่ `123456` → verify → เข้าระบบ

**สิ่งที่ระบบทำ:**
- สร้าง `auth.users` row
- Trigger สร้าง `drivers` row พร้อม user_id link
- KYC status = `unverified`

### ขั้นที่ 3: หน้า Home แสดง KYC banner (5 วินาที)

```
👋 ยินดีต้อนรับสู่ Favornoms Driver!

   ค่อยรับงานได้

   [อัปโหลดเอกสาร]
```

### ขั้นที่ 4: อัปโหลด KYC (3 นาที)

กดปุ่ม → ไปหน้า `/app/profile`

**5 รายการ:**

```
📄 บัตรประชาชน (ด้านหน้า)        [📷 Upload]
🚗 ใบขับขี่ (ด้านหน้า)            [📷 Upload]
🚗 ใบขับขี่ (ด้านหลัง)            [📷 Upload]
🪪 ทะเบียนรถ                      [📷 Upload]
🤳 รูป selfie ถือบัตรประชาชน      [📷 Upload]
```

ถ่ายรูปทีละใบ → กดอัปโหลด → ระบบเก็บใน Storage bucket `driver-kyc` (private)

### ขั้นที่ 5: รออนุมัติ (รอ admin)

KYC status → `pending` (รออนุมัติ)
หน้าจอแสดง: "รอ admin ตรวจสอบ (ประมาณ 1-2 ชั่วโมง)"

**🔄 ฝั่ง admin (พี่ก้อย):**
- เปิด `/b/<branchId>/drivers` → เห็นพี่เอกในรายชื่อ Pending
- กด "Review KYC" → ดูรูปทั้ง 5 → ทุกอย่างถูกต้อง
- กด **Approve** → call `set_driver_kyc_status(driver_id, 'verified')`

หลัง approve → พี่เอกเปิดแอพใหม่ → KYC verified ✅

# 🛠️ Developer / DevOps (สำหรับคนทำงาน Backend)

## บริบท
คุณคือทีมพัฒนา — ดูแลโครงสร้าง + แก้บัก + deploy ของใหม่

## งานประจำวัน

### เช้า: เช็ค error overnight

1. เปิด Sentry → ดู error ที่เกิดเมื่อคืน
2. ถ้ามีข้อมูล stack trace → ตามแก้
3. Sentry alert ถ้าตั้ง threshold ไว้

### Pull request review

```bash
git fetch
gh pr list
gh pr checkout 42
pnpm install
pnpm -r type-check
pnpm -r test
pnpm -r build
```

ตรวจดู changes → comment → approve

### Migration ใหม่

1. เขียน migration → `mcp__supabase__apply_migration`
2. หลัง DDL change → regenerate types
3. Commit migration file + types ด้วยกัน

### Edge function update

1. แก้ source ใน `supabase/functions/<name>/index.ts`
2. Deploy via `mcp__supabase__deploy_edge_function`
3. ดู logs ผ่าน MCP

### Incident response

มี Sentry alert "Order failed 500 errors x50":
1. เปิด Sentry → ดู stack trace
2. เปิด Supabase Edge fn logs ตามที่ runbook บอก
3. หาสาเหตุ → แก้ → deploy
4. Update `docs/RUNBOOK.md` ด้วยถ้าเจอ pattern ใหม่
